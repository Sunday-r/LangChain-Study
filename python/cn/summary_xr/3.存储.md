# LangChain：Memory（记忆）— 代码笔记

LLM 本身是无状态的，不记得历史对话。Memory 模块用于将先前对话嵌入到模型输入中，使其具备连续对话能力。本节介绍四种记忆模块。

---

## 一、ConversationBufferMemory（对话缓存记忆）

**特点：** 完整保存所有历史对话，不做任何截断。

### 基本用法

```python
from langchain.chat_models import ChatOpenAI
from langchain.chains import ConversationChain
from langchain.memory import ConversationBufferMemory

llm = ChatOpenAI(temperature=0.0)
memory = ConversationBufferMemory()
conversation = ConversationChain(
    llm=llm,
    memory=memory,
    verbose=False,   # True 可查看 LangChain 生成的完整提示
)
```

### 多轮对话

```python
conversation.predict(input="Hi, my name is Andrew")
conversation.predict(input="What is 1+1?")
conversation.predict(input="What is my name?")  # → "Your name is Andrew."
```

**要点：** 模型能回忆起第一轮提到的名字，因为所有对话都保存在 memory 中。

### 查看和手动操作记忆

```python
# 查看历史消息
print(memory.buffer)

# 另一种方式查看
memory.load_memory_variables({})  # 返回 {'history': 'Human: ...\nAI: ...'}

# 手动向缓存区添加对话
memory = ConversationBufferMemory()
memory.save_context({"input": "Hi"}, {"output": "What's up"})
memory.save_context({"input": "Not much"}, {"output": "Cool"})
memory.load_memory_variables({})
# → {'history': 'Human: Hi\nAI: What\'s up\nHuman: Not much\nAI: Cool'}
```

**总结：** `memory.buffer` 直接拿原始文本，`load_memory_variables({})` 返回字典格式。`save_context()` 可手动注入对话。**缺点：对话越长，token 越多，成本越高。**

---

## 二、ConversationBufferWindowMemory（对话缓存窗口记忆）

**特点：** 只保留最近 k 轮对话，丢弃更早的历史。

### 基本用法

```python
from langchain.memory import ConversationBufferWindowMemory

memory = ConversationBufferWindowMemory(k=1)  # 只保留最近 1 轮

memory.save_context({"input": "Hi"}, {"output": "What's up"})
memory.save_context({"input": "Not much"}, {"output": "Cool"})
memory.load_memory_variables({})
# → {'history': 'Human: Not much\nAI: Cool'}    ← 第一轮被丢弃了
```

### 应用到对话链

```python
memory = ConversationBufferWindowMemory(k=1)
conversation = ConversationChain(
    llm=llm,
    memory=memory,
    verbose=False,
)

conversation.predict(input="Hi, my name is Andrew")
conversation.predict(input="What is 1+1?")
conversation.predict(input="What is my name?")
# → "I'm sorry, I don't have access to that information."
# 因为 k=1，第三轮时第一轮的"我叫Andrew"已经被丢弃
```

**要点：** `k` 控制窗口大小。`k=1` 只记住上一轮，更早的信息丢失。适合只关心近期上下文的场景。

---

## 三、ConversationTokenBufferMemory（对话 token 缓存记忆）

**特点：** 不按轮数限制，而是按 **token 数量** 限制。超出时裁掉最早的部分。

```python
from langchain.memory import ConversationTokenBufferMemory

memory = ConversationTokenBufferMemory(llm=llm, max_token_limit=30)

memory.save_context({"input": "AI is what?!"}, {"output": "Amazing!"})
memory.save_context({"input": "Backpropagation is what?"}, {"output": "Beautiful!"})
memory.save_context({"input": "Chatbots are what?"}, {"output": "Charming!"})
memory.load_memory_variables({})
# → {'history': 'AI: Beautiful!\nHuman: Chatbots are what?\nAI: Charming!'}
# 最早的 "AI is what?! / Amazing!" 被 token 限制裁掉了
```

**要点：** 需要传入 `llm` 参数（用于计算 token 数），`max_token_limit` 控制上限。比窗口记忆更精细——按实际 token 开销控制。

---

## 四、ConversationSummaryBufferMemory（对话摘要缓存记忆）

**特点：** 用 LLM 对早期对话生成**摘要**，最近对话保留原文。兼顾压缩与信息保留。

```python
from langchain.memory import ConversationSummaryBufferMemory

schedule = """There is a meeting at 8am with your product team.
9am-12pm have time to work on your LangChain project.
At Noon, lunch at the italian resturant with a customer
who is interested in the latest in AI."""

memory = ConversationSummaryBufferMemory(llm=llm, max_token_limit=100)

memory.save_context({"input": "Hello"}, {"output": "What's up"})
memory.save_context({"input": "Not much, just hanging"}, {"output": "Cool"})
memory.save_context({"input": "What is on the schedule today?"}, {"output": f"{schedule}"})

memory.load_memory_variables({})
# → {'history': 'System: The human and AI engage in small talk before
#     the human asks about their schedule for the day. The AI informs ...'}
```

**要点：** 超出 token 限制后，早期对话被 LLM **压缩成摘要**（以 `System:` 开头），最近几轮保留原文。`max_token_limit` 控制 token 上限。

### 结合对话链使用

```python
conversation = ConversationChain(
    llm=llm,
    memory=memory,
    verbose=True,   # 可看到 Current conversation 中既有摘要又有原文
)

conversation.predict(input="What would be a good demo to show?")
# 模型能基于摘要中的日程信息回答问题
```

---

## 五、四种记忆对比总结

| 记忆类型 | 限制方式 | 优点 | 缺点 |
|---|---|---|---|
| `ConversationBufferMemory` | 无限制 | 信息完整 | token 开销大，成本高 |
| `ConversationBufferWindowMemory` | 按**轮数** (k) | 简单可控 | 粗粒度，可能丢失重要信息 |
| `ConversationTokenBufferMemory` | 按 **token 数** | 精细控制成本 | 仍直接丢弃早期对话 |
| `ConversationSummaryBufferMemory` | 按 **token 数** + 摘要 | 保留关键信息 | 需要 LLM 生成摘要，额外开销 |

**选择原则：** 短对话用 BufferMemory，控制成本用 WindowMemory 或 TokenMemory，长对话需要保留上下文用 SummaryBufferMemory。
