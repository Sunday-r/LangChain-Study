# LangChain：文档问答（Q&A with Documents）— 代码笔记

核心问题：LLM 单次只能处理有限 token，如何让它回答关于**大量文档**的问题？答案是通过 **Embedding + 向量存储 + 检索** 实现。

## 一、快速上手：VectorstoreIndexCreator（一行建索引）

```python
from langchain.indexes import VectorstoreIndexCreator
from langchain.vectorstores import DocArrayInMemorySearch
from langchain.document_loaders import CSVLoader

# 加载 CSV 文档
loader = CSVLoader(file_path="OutdoorClothingCatalog_1000.csv")

# 一行创建向量索引（内部自动完成：加载→分块→embedding→存入向量库）
index = VectorstoreIndexCreator(
    vectorstore_cls=DocArrayInMemorySearch
).from_loaders([loader])

# 查询
query = "Please list all your shirts with sun protection in a table in markdown"
response = index.query(query)
```

**要点：** `VectorstoreIndexCreator` 是最简封装，一行完成整个流程。`DocArrayInMemorySearch` 是纯内存向量存储，无需外部数据库。

## 二、拆解步骤：理解底层流程

上面的"一行"内部实际做了以下几步：

### 2.1 加载文档

```python
from langchain.document_loaders import CSVLoader

loader = CSVLoader(file_path="OutdoorClothingCatalog_1000.csv")
docs = loader.load()

docs[0]  # 每个元素是一个 Document 对象，包含 page_content 和 metadata
```

### 2.2 创建 Embedding

```python
from langchain.embeddings import OpenAIEmbeddings

embeddings = OpenAIEmbeddings()

# 对单条文本生成 embedding 向量（1536 维）
embed = embeddings.embed_query("Hi my name is Harrison")
print(len(embed))   # 1536
print(embed[:5])    # [-0.0219, 0.0067, -0.0182, ...]
```

**要点：** Embedding 将文本转为数值向量，语义相似的文本向量也相似，用于后续相似度检索。

### 2.3 存入向量数据库

```python
from langchain.vectorstores import DocArrayInMemorySearch

db = DocArrayInMemorySearch.from_documents(docs, embeddings)
```

**要点：** 将所有文档 embedding 后存入内存向量库。

### 2.4 相似度检索

```python
# 根据查询检索最相关的文档
docs = db.similarity_search("Please suggest a shirt with sunblocking")
print(len(docs))   # 返回最相关的 4 个文档
print(docs[0])     # 第一个结果确实是防晒衬衫
```

### 2.5 手动拼接文档 + 让 LLM 回答

```python
from langchain.chat_models import ChatOpenAI

llm = ChatOpenAI(temperature=0.0, max_tokens=1024)

# 将检索到的文档拼接成一个字符串
qdocs = "".join([docs[i].page_content for i in range(len(docs))])

# 手动构造提示让 LLM 回答
response = llm.call_as_llm(
    f"{qdocs} Question: Please list all shirts with sun protection in a table in markdown"
)
```

**要点：** 这是最原始的方式——手动拼文档 + 手动构造 prompt。LangChain 提供了链来封装这个过程。

## 三、RetrievalQA 链：封装检索 + 问答

```python
from langchain.chains import RetrievalQA

# 从向量库创建检索器接口
retriever = db.as_retriever()

# 创建检索问答链
qa_stuff = RetrievalQA.from_chain_type(
    llm=llm,
    chain_type="stuff",     # 指定链类型
    retriever=retriever,    # 检索器
    verbose=True,
)

# 一行查询
response = qa_stuff.run("Please list all your shirts with sun protection in a table in markdown")
```

**要点：** `RetrievalQA` 将"检索相关文档 → 拼接到提示 → LLM 生成回答"封装为一条链。

## 四、四种 chain_type 对比

当文档数量多、内容长时，不能简单把所有文档塞给 LLM，需要不同策略：

| chain_type | 工作方式 | 特点 |
|---|---|---|
| `stuff` | 所有文档拼成一个，一次调用 LLM | 简单快速，但文档过多会超 token 限制 |
| `map_reduce` | 每个文档单独问 LLM → 汇总所有回答 | 可并行，适合大量文档，但调用次数多 |
| `refine` | 逐个文档迭代，在前一个答案基础上优化 | 适合需要逐步累积信息的场景，速度较慢 |
| `map_rerank` | 每个文档单独打分 → 选最高分的回答 | 速度快但成本高，依赖 LLM 打分能力 |

## 五、整体流程图

```
CSV/PDF/网页 等文档
       ↓
  DocumentLoader（加载文档）
       ↓
  Text Splitter（分块，文档大时需要）
       ↓
  Embeddings（生成向量）
       ↓
  VectorStore（存入向量数据库）
       ↓
  similarity_search（相似度检索）
       ↓
  RetrievalQA（检索 + LLM 生成回答）
       ↓
  最终答案
```
